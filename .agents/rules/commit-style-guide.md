Comprehensive Commit Message Generation Instructions:
When generating commit messages, use the following Gitmoji mapping and rules to ensure a consistent, simple, and Title Case style:
1. Gitmoji Selection:
✨ :sparkles: — New features or functional additions.
🐛 :bug: — Bug fixes or logic corrections.
💄 :lipstick: — UI/UX improvements, CSS, and design adjustments.
♻️ :recycle: — Refactoring code (changing structure without changing behavior).
📝 :memo: — Documentation changes (README, comments, docs).
⚡️ :zap: — Performance improvements.
🔥 :fire: — Removing code or files.
🔧 :wrench: — Configuration changes (settings, .env, .gitignore).
➕ :heavy_plus_sign: — Adding new dependencies.
➖ :heavy_minus_sign: — Removing dependencies.
✅ :white_check_mark: — Adding or updating tests.
2. Formatting Rules:
Emoji First: Start every message with exactly one appropriate emoji.
Case: Use Title Case for the summary (e.g., "Updated Readme Instructions").
Mood: Use the Imperative Mood (e.g., "Add", "Fix", "Modify" instead of "Added", "Fixed", "Modified").
Simplicity: Keep it to one line. Combine related changes with "and" or commas.
Filenames: If a change is very specific to one file, mention it (e.g., 💄 Improved UI Design for "Header.tsx").
Examples:
♻️ Refactored Card Logic and Improved Code Readability
📝 Updated README with Project Overview and Installation Guide
🔧 Modified .gitignore to Exclude Local Environment Files
➕ Added Axios Dependency for API Requests
